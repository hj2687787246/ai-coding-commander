# 指挥官时间线

更新时间：2026-04-14

## 1. 文档定位

这份文档是指挥官体系的“时间线层”。

它只记录：

1. 会改变当前接班判断的完成事项
2. 会改变当前接班判断的重要节点

它不负责介绍项目总定位，也不负责高频问题解释。

如果是新窗口第一次接班，先读：

- `D:\Develop\Python-Project\Agent\docs\指挥官任命.md`
- `D:\Develop\Python-Project\Agent\docs\指挥官文档.md`

只有当需要快速知道“最近做完了什么”时，再读本文档。

---

## 2. 时间线使用规则

1. 只记会改变当前接班判断的节点
2. 不记一次性寒暄或零碎小问答
3. 新节点优先写“发生了什么、为什么重要、改变了什么”
4. 太旧且不再影响接班判断的节点，后续可以归档
