# Commander 目录说明

更新时间：2026-04-11

这个目录用于把当前仓库里的“指挥官系统”从项目专题文档中收拢出来，先在单仓内形成清晰分层，再决定哪些部分值得进一步独立复用。

当前迁移目标不是直接拆成新仓库，而是先把下列 5 层边界固定：

1. `core/`
   - 指挥官身份、职责、规则、生命周期
   - 这层回答“指挥官是谁、按什么规则工作”
2. `state/`
   - 当前任务卡、时间线、问题索引
   - 这层回答“这个项目当前做到哪、最近完成了什么、下一步是什么”
3. `outer/`
   - 新窗口交接、开窗模板、工作台、harness/Hermes-lite 说明
   - 这层是外圈配套，不是本体真相源
4. `transport/`
   - 执行模板、schema、调度/回收/恢复/提案等脚本
   - 这层是指挥官系统的手脚
5. `skill-source/`
   - 仓库内保存的 skill 真相源
   - 本地安装副本不是唯一来源

## 迁移原则

1. 先迁移，再留兼容壳，不直接硬删旧入口。
2. `core/` 和 `state/` 优先于 `outer/` 和 `transport/`。
3. 旧路径如果仍被现有脚本、模板或本地 skill 引用，先保留最小兼容层。
4. 项目业务事实、工程资料、业务代码仍留在仓库原位，不整体搬进 `commander/`。

## 当前目录职责

### `core/`

只放下列类型：

1. 任命
2. 主文档
3. 核心控制规则

### `state/`

只放下列类型：

1. 当前任务卡
2. 时间线
3. 问题索引

### `outer/`

只放下列类型：

1. 新窗口交接说明
2. 新窗口启动指令模板
3. agent workbench
4. 验收附件
5. harness / Hermes-lite 方案与实施说明

### `transport/`

只放下列类型：

1. 执行窗口模板
2. 结构化 schema
3. commander 调度、回收、恢复、检索、提案脚本

### `skill-source/`

只放仓库内维护的 skill 源文件，不放本地运行态数据。

## 兼容策略

迁移阶段旧路径会保留“兼容壳”：

1. Markdown 文档旧路径保留短跳转说明
2. 暂时仍被旧路径依赖的 prompt / script / schema，会先保留兼容入口
3. 等完成全量改引用并跑过至少两轮真实任务后，再统一删除旧壳

## 完成判据

这次迁移只有同时满足下面 4 条，才算真正收口：

1. `core/ state/ outer/ transport/ skill-source/` 都已经建立并装入对应资产
2. `AGENTS.md`、`README.md` 和主要入口都已经指向新路径
3. 至少两轮真实任务只靠新路径即可运行
4. 全仓搜旧路径时，只剩兼容壳和历史记录

## 压缩恢复锚点

1. `resume_anchor.json` 是压缩恢复的第一读入口，优先级高于 `checkpoint.json`。
2. `commander_resume.py --compact --task-id <task_id>` 应该只输出这个 compact 锚点，供新窗口快速恢复当前阶段、下一步和控制面决策。
3. `checkpoint.json` 只在需要更深状态、完整回放或排障时再读，不作为压缩恢复的默认首读文件。
