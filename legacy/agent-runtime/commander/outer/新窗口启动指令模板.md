# 新窗口启动指令模板

更新时间：2026-04-11

这份文档是给后续**新开窗口继续执行阶段计划**时直接复制用的。

目标只有一个：

**尽量减少上下文污染，让新窗口只拿到“当前阶段必须知道的信息”。**

它当前也应明确视为指挥官系统的外圈派工入口，而不是本体真相源：

- 它负责开窗协议和执行模板
- 不负责定义指挥官身份、长期记忆和当前主线真相
- 如果与 `commander/core/任命.md`、`commander/core/主文档.md` 冲突，始终以前两者为准

---

## 1. 使用原则

新窗口里不要再重复灌入整段历史，不要重新贴长篇背景，不要把所有文档一起塞进去。

每次新开窗口，只保留这 5 类信息：

1. 主项目路径
2. UTF-8 初始化要求
3. 总纲文档路径
4. 当前阶段文档路径
5. 本轮具体任务和验证要求

额外固定规则：

6. 如果本轮产生了代码或文档改动，并且验证通过，默认要在结束前整理并提交 git；不要停在“改完了但没提交”。
7. commit message 默认必须使用中文；如果我额外指定了前缀（例如 `4.6`），也必须带上这个前缀。
8. 只要本轮允许通过终端脚本写中文内容，提示词里必须显式加入编码安全规则，不能只写 UTF-8 初始化就算了。

这份模板现在默认也是指挥官的“开窗指挥协议”载体。也就是说：

- 指挥官不是临场随便写几句
- 而是默认按固定部件把任务压给新窗口
- 指挥官窗口本身默认不改业务代码；需要落地实现时，必须通过这里派给执行窗口

固定部件至少包括：

1. 项目路径
2. UTF-8 初始化
3. 编码安全规则
4. 新窗口角色
5. 必读文件
6. 本轮任务目标
7. 边界和禁区
8. 验证要求
9. git 规则
10. 汇报顺序

以后如果用户只说“帮我开个新窗口去做这件事”，指挥官默认就按这套协议生成提示词。

这套模板还有第二个作用：

**它不只是派工模板，也是结果回收模板。**

也就是说，指挥官之所以要把“汇报顺序”写死，不是形式主义，而是为了在新窗口做完后，能稳定接管结果并继续指挥下一步。

默认思路是：

1. 指挥官用固定部件派工
2. 新窗口按固定顺序回传结果
3. 指挥官据此判断：
   - 是否真的完成
   - 是否验证充分
   - 是否需要补读结果相关文件
   - 是否要继续拆下一轮任务
4. 指挥官默认先给结果分成：
   - `已收口`
   - `部分完成`
   - `阻塞待决策`

如果这次新开的不是执行开发窗口，而是知识问答 / 架构解释 / 学习规划窗口，不要直接套本文模板，优先使用：

## 4. 增强版执行原则（推荐按需附加）

如果你希望新窗口里的代理更重视“抓本质、走最短路径、少说废话”，可以在通用模板或阶段模板后面，再附加这一段。

这段比“只强调第一性原理”的原始写法更适合当前项目，因为它不会把代理过度推向空转讨论，而是要求它在约束明确后继续推进、落代码、跑验证。

```text
从问题本质和验收目标出发，不机械沿用惯例、模板或表面修补。

1. 如果目标、约束或验收标准存在关键歧义，并且会影响实现方向，先停下来确认；否则先按最小可验证路径推进，并明确你的假设。
2. 如果我给出的路径不是最短、最稳或风险最低的方案，直接指出，并给出更优替代方案与原因。
3. 遇到问题优先找根因，不做只掩盖现象的补丁；每个重要决策都要能回答“为什么这样做”。
4. 输出只保留会影响判断、实现和验收的关键信息，删掉不改变决策的内容。
5. 不要只停留在分析；在约束明确后，直接落代码、跑验证、再汇报。
```

适用场景：

- 需要抑制机械套模板
- 需要代理主动指出更短路径
- 需要减少“分析很多，但执行很少”的情况

不建议默认强加的场景：

- 你只是让新窗口做非常小的机械性改动
- 任务本身没有明显方案分歧
- 你更在意快速执行，而不是方案审视

## 17. skill / 复用工具沉淀窗口模板

适用场景：

- 某类重复问题已经多次出现，想判断该继续留在文档、补脚本，还是升级成 skill
- 想把外部 harness 的某个方法接入当前指挥官体系，但不复制平行骨架
- 想补一个本地 skill 雏形，同时明确它不能代替仓库 Markdown 真相源

```text
主项目：
D:\Develop\Python-Project\Agent

先按 UTF-8 规则初始化 PowerShell，再开始读文件：
[Console]::InputEncoding = [System.Text.UTF8Encoding]::new($false)
[Console]::OutputEncoding = [System.Text.UTF8Encoding]::new($false)
$OutputEncoding = [System.Text.UTF8Encoding]::new($false)
chcp 65001 > $null
$PSDefaultParameterValues['Get-Content:Encoding'] = 'utf8'
$PSDefaultParameterValues['Set-Content:Encoding'] = 'utf8'
$PSDefaultParameterValues['Add-Content:Encoding'] = 'utf8'

注意编码安全：凡是通过终端脚本写中文内容，不要直接把中文作为脚本字面量输出；优先用 Unicode 转义安全落盘。写完后必须做文件内容级校验，确认不是终端显示正常但文件实际写坏，也不是终端显示乱码但文件内容其实没问题。

你的角色：
你是执行窗口。本轮只负责处理“重复问题 -> 文档 / 脚本 / skill”的分流与最小落地，不重写指挥官体系，不改业务代码。

先读这些文件：
1. D:\Develop\Python-Project\Agent\README.md
2. D:\Develop\Python-Project\Agent\docs\agent_workbench.md

8. D:\Develop\Python-Project\Agent\prompts\execution_window_task_template.md

必须显式遵守：
1. 先读 md 文档真相源，再决定落到文档 / 脚本 / skill
2. 不要跳过文档直接写 skill
3. 不要把仓库事实抄进 skill 正文
4. 不复制 plans、memory、reports、debt 这类平行骨架

这次只执行：
- 只处理当前这类重复问题的沉淀载体选择
```

## 18. 不推荐的开窗方式

不要这样做：

- 把所有历史对话重新粘一遍
- 把所有阶段文档一次性塞进新窗口
- 让新窗口“顺便把后面几个阶段也一起看了”
- 只说“你继续上次的内容”，却不给当前阶段文档路径

这样最容易引入上下文幻觉。
