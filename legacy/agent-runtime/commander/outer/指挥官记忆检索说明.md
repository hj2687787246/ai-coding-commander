# 指挥官记忆检索说明
更新时间：2026-04-13

## 1. 这层是干什么的

这一层只做一件事：
给指挥官系统补一个 repo-native 的三层记忆检索入口。

一句话概括：

**它搜索现有真相源，不制造新的真相源。**

## 3. 现在怎么用

最常用命令：

```powershell
.\.venv\Scripts\python.exe -m commander.transport.scripts.commander_memory_search --query "memory retrieval layer"
```

只看某一层：

```powershell
.\.venv\Scripts\python.exe -m commander.transport.scripts.commander_memory_search --query "draft metadata" --layer session_runtime
```

只看 procedural memory：

```powershell
.\.venv\Scripts\python.exe -m commander.transport.scripts.commander_memory_search --query "worker report" --layer procedural_memory
```

如果要在别的工作区或本地技能目录下复用，可以显式指定：

```powershell
.\.venv\Scripts\python.exe -m commander.transport.scripts.commander_memory_search `
  --query "memory retrieval layer" `
  --repo-root . `
  --local-skill-root C:\Users\26877\.codex\skills
```

## 4. 返回结果怎么理解

返回结果是结构化 JSON，至少包含：

1. `searched_layers`
2. `searched_sources`
3. `index.layers`
4. `results`

其中 `index.layers` 会明确列出：

1. 每层索引了多少条 entry
2. 每层命中了多少条结果
3. 每层有哪些 source
4. 每层有哪些 source_kind

`results` 里的每条命中都会带：

1. `layer`
2. `source`
3. `source_kind`
4. `path`
5. `title`
6. `score`
7. `matched_lines`
8. `excerpt`

## 5. 边界

v1 仍然只做这些事：

1. 搜索
2. 返回可读路径和摘录
3. 让压缩后的上下文能恢复
4. 把 repo 脚本和本地 skills 当作可复用 procedural memory

v1 不做这些事：

1. 不建立第二套 plans / memory / reports / debt 平行骨架
2. 不把本地 skills 当成 repo facts
3. 不自动改 live skill
4. 不自动沉淀成独立数据库

一句话记忆：

**它是给现有真相源加一个可检索入口，而不是重写一套记忆系统。**
