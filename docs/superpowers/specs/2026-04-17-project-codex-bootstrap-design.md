# 项目内 `.codex` 指挥官协议设计

更新时间：2026-04-17

## 1. 背景

当前 `commander-mode` 已经能在 Agent 项目中恢复状态，但它仍然残留了 Agent 学习项目的特例恢复路径。结果是：

- 在当前 Agent 项目中运行时，它会回到学习模式相关文档；
- 在别的项目中运行时，如果该项目还没有自己的 `.codex` 协议骨架，`commander-mode` 也可能漂回 Agent 项目的学习状态语境；
- “学习模式”被误当成了系统默认模式，而不是当前项目里的一个任务模式。

本设计的目标，是把指挥官系统从 Agent 学习项目里抽离出来，做成一个**可移植的项目内 commander 协议**：

- 已初始化项目：恢复当前项目自己的状态；
- 未初始化项目：进入半自动初始化；
- 初始化后：围绕当前项目自己的任务模式继续生长；
- 学习模式只是任务模式之一，不再是全局默认。

## 2. 目标

### 2.1 目标

1. 让 `commander-mode` 能判断当前项目是否已经接入 commander 协议。
2. 对未初始化项目，提供半自动初始化流程。
3. 初始化时创建通用 `.codex` 骨架，而不是学习项目骨架。
4. 初始化后使用 `当前任务.md` 承载当前任务模式。
5. 支持至少 4 类通用任务模式：
   - 开发模式
   - 修复模式
   - 重构模式
   - 学习模式
6. 防止 `commander-mode` 再次漂回 Agent 项目的学习状态。

### 2.2 非目标

1. 本轮不做自动后台记忆提取。
2. 本轮不把指挥官系统做成完整平台产品。
3. 本轮不要求自动识别任务模式的复杂推理逻辑，第一版允许用户确认或简单推断。
4. 本轮不要求替换每个项目已经存在的自定义工作流，只定义最小可复用协议。

## 3. 三层结构

### 3.1 全局指挥官层

位置示意：

- `D:\Develop\Python-Project\ai-coding-commander\skills\commander-mode\SKILL.md`

职责：

- 识别当前工作区；
- 判断项目是否已初始化；
- 已初始化时走恢复；
- 未初始化时进入半自动初始化提示；
- 理解“任务模式”这个概念，但不内置任何项目专用模式内容。

### 3.2 仓库入口层

位置示意：

- `<repo>/AGENTS.md`

职责：

- 说明当前仓库身份；
- 说明第一跳入口；
- 说明敏感文件和边界；
- 指向项目自己的 `.codex` 协议。

### 3.3 项目内 `.codex` 协议层

位置示意：

- `<repo>/.codex/`

职责：

- 保存当前项目自己的恢复入口、当前状态、当前任务、验收、归档、偏好、周级审计等信息；
- 允许当前项目在其内部生长出不同任务模式的“血肉”。

## 4. 初始化判定规则

满足以下任一条件，就视为“已初始化项目”：

1. 存在 `.codex/AGENT.md`
2. 存在 `.codex/docs/当前状态.md`
3. 仓库根 `AGENTS.md` 明确把 `.codex/AGENT.md` 作为第一跳入口之一

如果上述条件都不满足，则视为“未初始化项目”。

## 5. 半自动初始化流程

### 5.1 已初始化项目

`commander-mode` 应该：

1. 读取仓库根 `AGENTS.md`
2. 读取项目 `.codex/AGENT.md`
3. 按项目自己的恢复入口继续
4. 识别当前任务模式
5. 恢复当前项目自己的状态

关键要求：

- 不再回到 Agent 项目的学习文档体系；
- 不再次创建 `.codex` 骨架。

### 5.2 未初始化项目

`commander-mode` 应该：

1. 明确说明当前项目还未接入 commander 协议；
2. 提出初始化建议，说明将创建最小 `.codex` 骨架；
3. 等待用户确认；
4. 用户确认后，执行初始化；
5. 初始化完成后进入当前任务模式判定。

关键要求：

- 不默认套用学习模式模板；
- 不自动无提示写入骨架。

## 6. 标准版最小 `.codex` 骨架

初始化完成后，项目内应至少具备：

```text
<repo>/
  AGENTS.md
  .codex/
    AGENT.md
    docs/
      当前状态.md
      当前任务.md
      恢复入口.md
      验收记录.md
      归档索引.md
      协作偏好.md
      周总结.md
```

### 6.1 各文件职责

- `.codex/AGENT.md`
  - 项目级 commander 工作手册
- `.codex/docs/当前状态.md`
  - 当前项目整体状态与当前结论
- `.codex/docs/当前任务.md`
  - 本轮正在推进的任务卡
- `.codex/docs/恢复入口.md`
  - 恢复优先级统一索引
- `.codex/docs/验收记录.md`
  - 已正式通过的任务记录
- `.codex/docs/归档索引.md`
  - 过期信息的归档与降级
- `.codex/docs/协作偏好.md`
  - 当前项目用户的稳定协作偏好
- `.codex/docs/周总结.md`
  - 周级总结与记忆审计入口

## 7. 当前任务与任务模式

### 7.1 为什么使用 `当前任务.md`

通用 commander 协议的默认骨架不应该直接长成学习项目结构，因此第一版使用通用的：

- `当前任务.md`

而不是默认创建：

- 学习进度卡
- 学习时间线
- 学习主线

### 7.2 `当前任务.md` 最少应包含

1. 当前任务名称
2. 当前任务目标
3. 当前任务模式
4. 当前进度
5. 当前卡点
6. 下一步
7. 最近验证

### 7.3 第一版支持的任务模式

1. 开发模式
2. 修复模式
3. 重构模式
4. 学习模式

原则：

- `commander-mode` 只理解“任务模式”这个概念；
- 每个项目自己决定模式的细节内容；
- 学习模式只是任务模式之一，不再是全局默认模式。

## 8. 实现形态

本设计需要同时落三类产物，不能只改一个 skill：

### 8.1 通用 commander 内核

位置：

- `skills/commander-mode/SKILL.md`

职责：

- 识别已初始化 / 未初始化
- 分流到恢复或初始化流程

### 8.2 标准 `.codex` 模板

位置建议：

- `skills/commander-mode/references/project-codex-layout.md`
- `skills/commander-mode/references/templates/project-codex-standard/`

职责：

- 定义新项目初始化后的标准骨架

### 8.3 初始化脚本

位置建议：

- `skills/commander-mode/scripts/bootstrap_codex_workspace.py`

职责：

- 在用户确认后，向当前项目写入最小 `.codex` 骨架

## 9. 验收矩阵

### 场景 A：已初始化项目恢复

测试对象：

- 当前 Agent 项目

预期：

1. `commander-mode` 判断项目已初始化
2. 按当前项目自己的 `.codex` 协议恢复
3. 不进入首次初始化流程

通过信号：

- 不提示“当前项目未初始化”
- 不建议重建 `.codex`
- 恢复结果围绕当前项目自己的状态与任务展开

### 场景 B：未初始化项目首次接入

测试对象：

- 一个没有 `.codex` 的全新项目目录

预期：

1. `commander-mode` 判断项目未初始化
2. 提示进入 commander 协议初始化
3. 等待用户确认
4. 确认后创建通用 `.codex` 骨架

通过信号：

- 不再回到 Agent 学习项目状态
- 创建的是通用骨架
- 默认文档是 `当前任务.md`，不是学习进度卡

### 场景 C：已初始化新项目二次进入

测试对象：

- 已经完成首次初始化的新项目

预期：

1. `commander-mode` 判断项目已初始化
2. 不重复创建骨架
3. 直接恢复该项目自己的状态

通过信号：

- 不再提示首次初始化
- 不重复创建 `.codex` 文件
- 读取并继续该项目自己的当前任务

### 场景 D：任务模式中性

测试对象：

- 新初始化项目

预期：

1. 默认存在 `当前任务.md`
2. 其中包含 `当前任务模式`
3. 任务模式默认为中性判定或待确认，不直接落学习模式

通过信号：

- 没有自动生成学习类专用文档
- 没有默认进入学习语境

### 场景 E：防串台

测试对象：

- 在非 Agent 项目中运行 `commander-mode`

预期：

1. 不再引用 Agent 项目的学习文档体系
2. 不再漂回 Week 1 / Week 2 / 学习主线表达

通过信号：

- 不提 `.codex/docs/学习进度卡.md`
- 不提 `.codex/docs/学习时间线.md`
- 不提 Agent 项目的学习阶段术语

## 10. 风险与边界

1. 如果项目已经存在自己的 `.codex` 但结构与标准模板不同，第一版应优先尊重现有项目结构，而不是强行重写。
2. 第一版允许初始化后的任务模式判定依赖用户确认，不要求全自动推断。
3. 如果仓库根 `AGENTS.md` 已经存在，不应粗暴覆盖，只能补最小入口说明或保持不动。

## 11. 一句话总结

`commander-mode` 以后应该先判断当前项目是否已初始化；未初始化时走半自动初始化，初始化后写入通用 `.codex` 骨架，并通过 `当前任务.md` 承载项目自己的任务模式。学习模式只是任务模式之一，不再是系统默认模式。
