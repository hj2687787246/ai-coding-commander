# 项目 `.codex` 工作手册

## 作用

- 定义当前项目的 commander 恢复顺序
- 约束当前项目的任务推进、验收和归档
- 说明项目内文档分工

## 恢复顺序

1. `.codex/docs/恢复入口.md`
2. `.codex/docs/当前状态.md`
3. `.codex/docs/当前任务.md`
4. `.codex/docs/协作偏好.md`

## 文档分工

- `当前状态.md`：当前项目整体状态
- `当前任务.md`：本轮正在推进的任务
- `验收记录.md`：已正式完成的事项
- `归档索引.md`：已过期或已降级内容
- `协作偏好.md`：当前项目用户的稳定协作偏好
- `周总结.md`：周级总结与轻量记忆审计

## 任务治理

- `当前任务.md` 是当前任务的主真相源。
- 当前任务至少应写清：任务模式、任务形状、执行强度、验证状态、验证证据。
- 未验证不得标记任务完成；没有验证证据时，当前任务不得视为完成。
- 长任务恢复时，优先读取磁盘上的当前任务真相源，不依赖聊天记忆。

## 写回纪律

- 完成有意义的工作后，优先更新 `当前任务.md`：
  - 当前进度
  - 下一步
  - 验证状态
  - 验证证据
- 当前项目整体结论变化时，更新 `当前状态.md`。
- 事项正式完成并有证据时，再写入 `验收记录.md`。
- 事项退出当前主线、过期或降级时，更新 `归档索引.md`。
- 不要把长聊天记录直接复制进项目文档；聊天只能作为线索，项目文档才是恢复真相源。
