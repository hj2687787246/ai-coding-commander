# 协作偏好

## 使用方式

- 只记录当前项目用户稳定、长期有效、会影响未来行为的协作偏好。
- 不记录一次性的聊天偏好、临时任务范围或聊天原文。
- 每条偏好应写成结构化记忆卡，便于 commander 按触发场景激活。

## Stable Preferences

### pref-example-verify-before-claim

```yaml
type: preference
status: stable
scope: project
triggers:
  - completion
  - validation
rule: 下结论前必须绑定新鲜验证证据。
do:
  - 运行相关测试、自检或 stop gate
  - 汇报具体命令和结果
dont:
  - 使用“基本好了”“应该可以”代替证据
evidence:
  - 示例卡片，初始化后可替换为当前项目真实偏好
```

## Candidate Preferences

候选偏好用于记录刚出现但尚未确认长期稳定的协作方式。再次出现或用户确认后，再升级到 `Stable Preferences`。
