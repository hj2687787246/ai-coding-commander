# 协作偏好

## 使用方式

- 只记录当前项目用户稳定、长期有效、会影响未来行为的协作偏好。
- 不记录一次性的聊天偏好、临时任务范围或聊天原文。
- 每条偏好应写成结构化记忆卡，便于 commander 按触发场景激活。

## Stable Preferences

### pref-token-roi

```yaml
type: preference
status: stable
scope: project
triggers:
  - planning
  - context_recovery
  - documentation
rule: token 使用目标是高价值，不是单纯低消耗。
do:
  - 优先读取能改变决策的真相源
  - 把输出压缩为结论、风险、下一步
dont:
  - 为了保险机械读取所有模板或完整历史
  - 把低 token 当成唯一优化目标
evidence:
  - 2026-04-28 用户明确纠正：不是低 token，而是 token 花得有价值
```

### pref-auto-memory

```yaml
type: preference
status: stable
scope: project
triggers:
  - handoff
  - long_task
  - memory
  - completion
rule: skill 应自动沉淀有恢复价值的状态和偏好，不等待用户说“沉淀”。
do:
  - 出现恢复价值节点时写入最小 checkpoint 或偏好卡
  - 把稳定偏好写成结构化卡片
dont:
  - 只在用户显式要求时才写回
  - 把聊天原文当成记忆
evidence:
  - 2026-04-28 用户明确要求 skill 自动运行、自动沉淀
```

### pref-long-task-checkpoint

```yaml
type: preference
status: stable
scope: project
triggers:
  - long_task
  - implementation
  - interruption
  - handoff
rule: 长任务必须过程型 checkpoint，避免中断后状态回到开始前。
do:
  - 关键阶段完成后覆盖式更新当前任务
  - 等待长命令或可能中断前写明当前目标、进度和下一步
dont:
  - 只在收尾时总结
  - 让当前任务文件停留在旧阶段
evidence:
  - 2026-04-28 用户指出长任务状态不会实时更新且中断后记忆回到开始前
```

### pref-skill-not-platform

```yaml
type: preference
status: stable
scope: project
triggers:
  - architecture
  - planning
  - product_direction
rule: 这个项目主要作为可复用 skill 使用，不做平台，也不强制用户按模板工作。
do:
  - 优先增强 commander-mode skill 本体和可选记忆面
  - 保持 .codex 模板为可选增强包
dont:
  - 把路线设计成平台化 runtime
  - 要求所有仓库必须套模板才能使用
evidence:
  - 2026-04-28 用户明确说明主要把项目当 skill 使用，并非平台
```

### pref-no-repeat-habits

```yaml
type: preference
status: stable
scope: project
triggers:
  - conversation
  - context_recovery
  - memory
rule: 用户不希望每次重复说明协作习惯，skill 应从本地偏好卡激活相关习惯。
do:
  - 恢复时读取协作偏好并选择本轮适用偏好
  - 收尾前检查是否有新偏好需要写回
dont:
  - 依赖聊天长记忆
  - 让用户反复重申同一协作方式
evidence:
  - 2026-04-28 用户明确说希望 skill 根据用户习惯沉淀，避免每次重复说明
```

### pref-executable-rules

```yaml
type: preference
status: stable
scope: project
triggers:
  - skill_design
  - rules
  - testing
rule: 规则不能只堆在文档里，关键习惯要有触发器、gate 或测试来提高执行概率。
do:
  - 为重要规则设计触发条件和收尾检查
  - 能测试的行为用测试锁定
dont:
  - 继续写长篇规则但不绑定执行时机
  - 把“读过规则”误当成“会执行规则”
evidence:
  - 2026-04-28 用户担心 agent 读了偏好和 AGENT.md 也不会执行
```
